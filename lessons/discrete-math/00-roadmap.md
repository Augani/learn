# Discrete Math & Logic — Track Roadmap

> The mathematical backbone of computer science.
> Everything your compiler, database, and crypto library secretly depend on.

## Why This Track?

Every algorithm you write rests on discrete math. When you write `if x > 0 and not done`,
you're doing propositional logic. When you analyze Big-O, you're using recurrence relations.
When you trust HTTPS, you're trusting number theory. This track makes the implicit explicit.

## Prerequisites

- Basic programming (Python examples throughout)
- Comfort with variables and functions
- No prior math background beyond algebra

## How to Use This Track

1. **Go in order** — each lesson builds on the previous
2. **Do the exercises** — math is a contact sport
3. **Write code** — Python examples make concepts concrete
4. **Use the references** — keep notation and proof sheets open

## Time Estimate

- ~30 minutes per lesson
- ~7-8 hours total for the full track
- Exercises add 15-20 minutes each

## What You'll Be Able to Do After

- Read and write formal proofs
- Analyze algorithm complexity from recurrences
- Understand the math behind RSA encryption
- Know why regex can't parse HTML (formally!)
- Recognize NP-complete problems in the wild

## Recommended Reading

These books are optional — the lessons above cover everything you need. But if you want to go deeper:

- **Discrete Mathematics and Its Applications** by Kenneth Rosen (McGraw-Hill, 8th Edition 2019) — The standard textbook
- **Mathematics for Computer Science** by Eric Lehman, F. Thomson Leighton, and Albert Meyer (MIT) — Rigorous and comprehensive. *Free from MIT OpenCourseWare*
