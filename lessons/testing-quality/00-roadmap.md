# Testing & Quality: Complete Learning Path

> **Why this topic matters**: Software that isn't tested is software you
> can't trust. Testing isn't about finding bugs — it's about building
> confidence that your code works today and will keep working tomorrow.
> Every professional developer needs these skills.

## How to Use This Path

**Total estimated time**: 30-40 hours

**If you're brand new to testing**: Start at lesson 01 and go in order.
Every lesson has code examples you should type out and run yourself.

**If you already write some tests**: Skim lessons 01-04, then dive into
05 (mocking) and 08 (TDD) which are where most developers level up.

**If you're preparing for a job**: Focus on lessons 03, 05, 06, 08, and
the capstone (18). These cover what interviewers ask about most.

---

## What You'll Build

By the end of this path, you'll be able to:

1. Write unit, integration, and end-to-end tests confidently
2. Use TDD to design better code
3. Know when to mock and when not to
4. Set up CI pipelines that catch bugs before production
5. Measure and improve test quality with coverage and mutation testing
6. Test tricky things: async code, databases, external APIs

## Recommended Reading

These books go deeper than any tutorial. If you can access them through
a library or affordable edition, they're worth your time.

| Book | Author | Why Read It |
|------|--------|-------------|
| **Unit Testing Principles, Practices, and Patterns** | Vladimir Khorikov (Manning, 2020) | The best modern guide to writing tests that actually help. Covers what makes a good test vs a fragile one. |
| **Test Driven Development: By Example** | Kent Beck (Addison-Wesley, 2002) | The original TDD book by its creator. Walks through TDD step-by-step with real examples. Timeless. |
| **Growing Object-Oriented Software, Guided by Tests** | Steve Freeman and Nat Pryce (Addison-Wesley, 2009) | Shows how testing shapes software design. Essential for understanding the London school of TDD. |

## Prerequisites

You should be comfortable with:
- Writing functions in at least one programming language
- Basic command-line usage (running programs, navigating directories)
- What a function return value is

No prior testing experience needed. We start from zero.
