# Design Patterns: Building Software That Lasts

> **Why this topic matters**: Design patterns are proven solutions to
> problems that show up again and again in software. Learning them is
> like learning chess openings — you don't have to reinvent strategy
> from scratch every game. They give you a shared vocabulary with
> other developers and a toolkit for writing code that's easier to
> change, test, and understand.

---

## Who This Is For

You should be comfortable writing code in at least one language. You
don't need to be an expert — if you can write functions, use classes
or structs, and understand basic inheritance or interfaces, you're
ready.

## How to Use This Course

**If you're brand new to patterns**: Go in order. Each lesson builds
on previous ones. Don't skip the SOLID lessons — they're the
foundation everything else rests on.

**If you know some patterns already**: Jump to whatever you need.
Each lesson is self-contained enough to read on its own.

**For interview prep**: Focus on lessons 01-11, plus the reference
cards. Most interview questions test GoF patterns and SOLID.

**For real-world projects**: Lessons 12-16 on architecture patterns
are what separate hobby projects from production systems.

## Recommended Reading

These books shaped how the industry thinks about software design.
This course covers the core ideas, but if you want to go deeper:

- **Design Patterns: Elements of Reusable Object-Oriented Software**
  by Gamma, Helm, Johnson, Vlissides (Addison-Wesley, 1994) — "The
  Gang of Four Book." The original. Dense but definitive. Every
  pattern in lessons 04-11 comes from this book.

- **Head First Design Patterns** by Eric Freeman and Elisabeth Robson
  (O'Reilly, 2nd Edition 2020) — The most approachable introduction.
  Visual, example-heavy, great for beginners. If you buy one book,
  make it this one.

- **A Philosophy of Software Design** by John Ousterhout (Yaknyam
  Press, 2nd Edition 2021) — Not about patterns specifically, but
  about the deeper skill of managing complexity. Short and excellent.

- **Clean Architecture** by Robert C. Martin (Pearson, 2017) — The
  ideas behind lessons 13-14. How to structure large systems so
  they survive years of change.
