# CS Fundamentals — No Degree Required

Everything a self-taught programmer needs to know about how computers
actually work. No jargon without explanation. Every concept explained
with everyday analogies first, then the technical details.

This track is language-agnostic but uses Rust, Go, and Python examples
where helpful. It's designed to fill the gaps that tutorials skip over.

## Who This Track Is For

- Self-taught developers who skipped a CS degree
- Bootcamp grads who want to understand "the why"
- Anyone who feels shaky on fundamentals but can already write code
- People learning Rust, Go, or systems programming who keep hitting unfamiliar concepts

## What This Track Is NOT

- Not a textbook. No proofs, no academic exercises.
- Not language-specific. The concepts apply everywhere.
- Not about algorithms and data structures (that's its own track).

---

## After This Track

You'll understand what's happening "under the hood" when you write code
in any language. You'll know why Rust has ownership, why Go has a garbage
collector, why JavaScript is single-threaded, and why C programmers
worry about buffer overflows.
